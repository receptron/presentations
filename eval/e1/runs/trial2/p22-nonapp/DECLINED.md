# Not a collection request

"Write me a poem about databases" is a one-off creative writing request, not a
data-management need. There is no data model, no records to track over time,
and no UI the user wants to revisit — so authoring a schema.json would be
inappropriate scaffolding.

How I would actually respond: write the poem directly in chat. For example, a
short piece playing on database concepts (tables, joins, transactions, ACID,
normalization) — e.g. rows as memories, JOINs as reunions, ROLLBACK as regret,
COMMIT as promises kept — and offer to adjust tone, length, or style if the
user wants a different take.

No files beyond this note should be created for such a request.
