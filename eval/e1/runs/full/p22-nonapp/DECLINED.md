# Declined: not a collection request

"Write me a poem about databases" is a one-off creative writing request, not a
data-collection request. There is no recurring data model — no records, fields,
relations, views, or lifecycle to declare in a `schema.json`. Authoring a
collection here would add machinery the user never asked for.

How I would actually respond: write the poem directly in the chat reply
(e.g. a short piece playing on tables, joins, transactions, and the irony that
"the workspace is the database"). If the user later wanted to *keep* a growing
set of poems, I would then offer a simple `poems` collection — but only on
request.
